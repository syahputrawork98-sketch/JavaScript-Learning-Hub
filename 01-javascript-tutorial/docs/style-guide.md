# Style Guide - 01 JavaScript Tutorial

## Tujuan Style Guide
Menjaga materi tetap detail, ringkas, mudah dipahami, dan konsisten antar bab.

## Aturan Bahasa
- Pakai bahasa Indonesia sehari-hari.
- Istilah teknis boleh dipakai, tapi langsung dijelaskan singkat.
- Hindari paragraf panjang; utamakan poin-poin terstruktur.
- Hindari kata-kata kabur seperti "pokoknya" atau "intinya gitu".

## Aturan Penjelasan
- Urutan wajib: kenapa -> bagaimana -> contoh -> latihan.
- Tiap klaim teknis harus bisa diuji lewat kode.
- Selalu jelaskan sebab-akibat, bukan hanya hasil akhir.

## Aturan Contoh Kode
- Contoh harus minimal tapi cukup menjelaskan konsep.
- Sertakan prediksi output sebelum output aktual.
- Untuk topik rawan salah, tampilkan contoh salah dan perbaikannya.

## Aturan Latihan
- Bertingkat: dasar, menengah, menantang.
- Soal harus menguji pemahaman, bukan hafalan sintaks.
- Minimal 1 soal debugging per bab.

## Standar Visual Wireframe (Ditetapkan)
- Format visual yang dipakai adalah gaya hybrid: profesional + friendly.
- Emoji dipakai secukupnya pada judul node/section utama.
- Kontras tinggi, alur panah jelas, dan hierarchy mudah dibaca.
- Pertahankan konsistensi gaya pada file berikut:
- `../assets/learning-flow.svg`
- `../assets/chapter-layout.svg`

## Definition of Done Materi Bab
- Struktur mengikuti `chapter-template.md`.
- Scope sesuai `content-map.md`.
- Checkpoint lulus bisa diverifikasi.
- Tidak overlap mendalam dengan track 02.
