# Aturan Kontribusi dan Perubahan Repository

Dokumen ini menjelaskan aturan ketika menambah, mengubah, memindahkan, atau merapikan isi repository.

Tujuannya adalah menjaga agar repository tetap konsisten sebagai **rak buku JavaScript** yang terorganisir dengan baik.

Repository ini dirancang sebagai **knowledge base jangka panjang tentang JavaScript**, sehingga setiap perubahan harus menjaga kualitas materi dan struktur repository.

---

# Prinsip Umum

Setiap perubahan pada repository harus memperkuat salah satu dari dua hal berikut:

- kualitas materi dalam buku
- kerapian struktur repository

Perubahan tidak boleh membuat repository kehilangan fokus utamanya sebagai **referensi pembelajaran JavaScript**.

---

# Penambahan Materi Baru

Sebelum menambahkan materi baru, pastikan bahwa:

- topik benar-benar relevan dengan JavaScript
- topik ditempatkan pada buku yang tepat
- topik tidak menduplikasi penjelasan yang sudah ada
- topik membantu memperjelas konsep JavaScript

Materi tidak boleh ditambahkan hanya karena topik tersebut populer.

Prioritas utama adalah **konsep fundamental JavaScript**.

---

# Penempatan Materi

Aturan penempatan materi dalam repository:

- materi JavaScript harus berada di dalam folder buku
- aturan repository harus berada di dalam folder `docs`
- `README.md` di root hanya menjelaskan gambaran besar repository
- penjelasan konsep teknis tidak boleh ditempatkan di folder `docs`

Dengan aturan ini, struktur repository tetap jelas antara:

- **materi pembelajaran**
- **aturan repository**

---

# Perubahan Struktur Repository

Jika ingin mengubah struktur repository, beberapa hal berikut harus diperhatikan:

- perubahan tidak boleh merusak konsep repository sebagai rak buku
- domain pembahasan setiap buku harus tetap jelas
- perubahan tidak boleh mencampur domain pembahasan antar buku

Perubahan struktur sebaiknya dilakukan hanya jika benar-benar diperlukan.

---

# Revisi Materi

Ketika merevisi materi yang sudah ada, fokus utama adalah:

- memperbaiki akurasi konsep
- memperjelas penjelasan
- memperbaiki contoh kode
- menghapus bagian yang berpotensi menimbulkan miskonsepsi

Perubahan tidak boleh mengurangi kedalaman penjelasan materi.

---

# Konsistensi Repository

Setiap perubahan sebaiknya menjaga konsistensi pada:

- gaya bahasa
- struktur penjelasan
- penggunaan istilah teknis
- tingkat kedalaman materi

Konsistensi ini penting agar seluruh repository terasa seperti **satu kumpulan buku yang saling terhubung**.

---

# Hal yang Perlu Dihindari

Beberapa hal berikut sebaiknya dihindari dalam perubahan repository:

- menambahkan materi framework yang tidak relevan
- mencampur beberapa domain JavaScript dalam satu pembahasan
- menambahkan materi yang terlalu dangkal
- menambahkan contoh kode yang tidak menjelaskan konsep dengan jelas
- menambahkan topik yang keluar dari fokus JavaScript sebagai bahasa dan runtime

---

# Tujuan Akhir

Tujuan dari aturan ini adalah menjaga repository agar tetap menjadi **referensi JavaScript yang kuat, rapi, dan dapat dibaca ulang dalam jangka panjang**.

Setiap perubahan sebaiknya membantu repository ini berkembang sebagai **sumber pemahaman JavaScript yang mendalam**, bukan sekadar kumpulan catatan belajar.
