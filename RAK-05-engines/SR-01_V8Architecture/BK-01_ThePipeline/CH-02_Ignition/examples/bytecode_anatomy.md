# Lab: Bytecode Anatomy (Reading Ignition)

Ignition adalah interpreter berbasis **register** yang menggunakan **Accumulator** untuk sebagian besar operasinya.

## 🛠️ Cara Melihat Bytecode
Anda bisa menggunakan flag V8 di Node.js untuk melihat bytecode yang dihasilkan:

```powershell
node --print-bytecode -e "function add(a, b) { return a + b; } add(1, 2);"
```

## 🔍 Bedah Bytecode `add(a, b)`

Misalkan kita memiliki fungsi:
```javascript
function add(a, b) {
  return a + b;
}
```

V8 akan menghasilkan bytecode seperti ini (disederhanakan):

```bytecode
[generated bytecode for function: add]
Parameter count 3 (termasuk .this)
Register count 0
   0 : Ldar a0          // Load Accumulator dari register a0 (parameter a)
   2 : Add a1, [0]      // Tambahkan nilai dari register a1 (parameter b) ke Accumulator
   5 : Return           // Kembalikan nilai yang ada di Accumulator
```

### 📋 Glosarium Singkat
1. **Ldar (Load Accumulator Register)**: Mengambil nilai dari register dan menaruhnya di Accumulator.
2. **Star (Store Accumulator Register)**: Mengambil nilai dari Accumulator dan menyimpannya ke register.
3. **Accumulator**: Register khusus tempat hasil operasi aritmatika/logika disimpan sementara.

## 🧠 Mental Model
Bayangkan Ignition sebagai **Koki di Dapur Kecil**. 
- **Accumulator** adalah "Piring Utama" di depannya. 
- Bahan makanan (data) diambil dari rak (Registers), ditaruh di piring (Accumulator), diolah (Add/Sub), lalu disajikan (Return).
