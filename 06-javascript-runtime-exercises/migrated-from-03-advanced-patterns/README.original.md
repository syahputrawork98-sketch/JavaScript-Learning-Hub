# JavaScript Advanced Patterns

Track ini fokus pada pengambilan keputusan teknis ketika kode mulai besar dan kompleks.

## Tujuan
- Memilih pattern yang tepat untuk maintainability.
- Mendesain alur async dan error handling yang stabil.
- Menjaga keterbacaan sambil mengontrol risiko regresi.

## Scope
- Modul dan boundary desain kode.
- Error strategy dan defensive coding.
- Async orchestration (sequential, parallel, fail-fast vs fail-soft).
- Pattern refactor untuk codebase menengah.

## Out of Scope
- Framework-specific architecture.
- Micro-optimization tanpa alasan bisnis/teknis yang jelas.

## Exit Criteria
- Bisa menjelaskan alasan teknis di balik pattern yang dipilih.
- Bisa refactor alur async/error tanpa mengubah perilaku utama.
