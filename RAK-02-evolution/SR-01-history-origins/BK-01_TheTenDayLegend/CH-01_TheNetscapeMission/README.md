# CH-01: The Netscape Mission (1995)

**"Misi Perekat Web Statis"**
*Target: Memahami konteks lahirnya JavaScript dalam waktu < 2 menit.*

## 1. Definisi & Konsep (The Logic)
Pada era awal web (awal 1990-an), halaman web bersifat sepenuhnya statis. Setiap interaksi kecil mengharuskan pemuatan ulang halaman dari server. Netscape Communications menyadari perlunya sebuah "bahasa perekat" (glue language) untuk memberikan interaktivitas langsung di sisi klien.

### Terminologi Utama (Senior Terms)
- **Static Web Crisis**: Kondisi di mana web kekurangan dinamisme dan interaktivitas.
- **Client-Side Scripting**: Eksekusi skrip langsung di browser pengguna, bukan di server.
- **Glue Language**: Bahasa yang dirancang untuk menghubungkan berbagai komponen perangkat lunak.

## 2. Rasionalitas (Why & How?)
JavaScript diciptakan untuk menyelesaikan kesenjangan antara desainer web (yang butuh sesuatu yang mudah) dan programmer berat (yang menggunakan Java/C++). 

### Analogi Mendalam
Bayangkan sebuah rumah yang semua furniturnya dipaku ke lantai (Web Statis). Jika Anda ingin menggeser kursi, Anda harus meruntuhkan rumah itu dan membangunnya kembali dari awal. JavaScript adalah roda yang dipasang pada furnitur tersebut, memungkinkan Anda menggesernya secara dinamis tanpa merusak fondasi rumah.
